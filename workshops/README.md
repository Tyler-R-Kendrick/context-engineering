# Workshop Exercises

This directory contains hands-on exercises to practice context engineering techniques with GitHub Copilot.

## Overview

These workshops are designed as a progressive learning path, focusing first on prompt engineering fundamentals, then advancing to tools and patterns.

## Exercises

### 1. Improving Vague Prompts (15 minutes)
**File:** `exercise-01-improving-prompts.md`

Learn to transform vague prompts into specific, actionable ones.

**Topics:**
- Specificity and clarity
- Input/output specifications
- Constraints and requirements
- Edge case handling

**Skills:** Prompt writing, requirement analysis

---

### 2. Context Through Comments (20 minutes)
**File:** `exercise-02-context-comments.md`

Master using comments to provide context for code generation.

**Topics:**
- Strategic comment placement
- Requirement documentation
- Error handling specifications
- Performance constraints

**Skills:** Documentation, context provision

---

### 3. Test-Driven Context (25 minutes)
**File:** `exercise-03-test-driven-context.md`

Use tests as specifications to drive implementation.

**Topics:**
- Writing tests first
- Defining behavior through tests
- Edge case discovery
- Test-driven development

**Skills:** Testing, specification, TDD

---

### 4. Tools and Context Engineering (30 minutes)
**File:** `exercise-04-tools-and-context.md`

Explore various tools and techniques for enhancing context.

**Topics:**
- Type systems as context
- Documentation files
- Configuration files
- Pattern files
- Combining multiple tools

**Skills:** Tool integration, comprehensive context engineering

---

## How to Use These Workshops

### Individual Learning
1. Start with Exercise 1 and progress sequentially
2. Complete all examples in each exercise
3. Reflect on the questions provided
4. Apply learnings to your own projects
5. Track your progress and improvements

### Team Workshops
1. Allocate 90-120 minutes total
2. Work through exercises together
3. Share and compare solutions
4. Discuss best practices as a team
5. Create team standards based on learnings

### Self-Paced Learning
1. Complete one exercise per day/week
2. Practice techniques on real work
3. Return to review and refine
4. Build a personal toolkit
